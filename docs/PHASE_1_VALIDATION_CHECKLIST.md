# Phase 1 Validation Checklist

**For Use During User Testing**

---

## Pre-Test Setup

- [ ] Production deployed and accessible
- [ ] Test users identified (5-7 real users)
- [ ] Testing environment ready
- [ ] Recording method ready (notes, screen recording, etc.)
- [ ] Validation questions prepared

---

## During Test (First 60 Seconds - Silent Observation)

- [ ] User lands on `/` (public unstuck entry)
- [ ] User sees: "What do you need help with?"
- [ ] User types question/request
- [ ] User receives one clear next action
- [ ] User sees login prompt: "Want to save this and get the next step when you come back?"
- [ ] Note: Any confusion or hesitation observed
- [ ] Note: Time to first action (target: < 60 seconds)
- [ ] Note: When login prompt appears (should be after action delivered)

---

## Post-Test Questions

## Pass/Fail Assessment

**Mark each criterion:**
- [ ] ✅ PASS: User articulated the next action
- [ ] ✅ PASS: User reported reduced overwhelm
- [ ] ✅ PASS: Login felt like a benefit, not a barrier

**Overall Result:**
- [ ] ✅ PASS (all criteria met)
- [ ] ❌ FAIL (one or more criteria not met)

---

## Next Steps

**If PASS:**
- [ ] Document results in `PHASE_1_VALIDATION_TRACKING.md`
- [ ] Proceed to Phase 2 planning

**If FAIL:**
- [ ] Document specific failure points
- [ ] Identify adjustments needed (copy/timing/cues only)
- [ ] Plan re-test
